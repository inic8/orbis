# Postprocessing

This directory contains the teacher-student recovery workflow for finetuning a pruned Orbis world model.

The setup assumes:
- the original teacher model lives in `logs_wm/orbis_288x512`
- the pruned student model lives in `logs_wm/orbis_288x512_pruned`
- the tokenizer lives in `logs_tk/tokenizer_288x512`

Those defaults are resolved inside `student_teacher_recovery.py`, so the configs in this directory do not need to repeat them unless you want to override them.

## Files

- `student_teacher_recovery.py`: teacher-student Lightning module used for recovery training.
- `prepare_video_dataset.py`: converts videos under `postprocessing/data` into HDF5 datasets and manifests compatible with the repo dataloaders.
- `configs/teacher_student_recovery.yaml`: generic teacher-student recovery config.
- `configs/teacher_student_recovery_v100.yaml`: V100-oriented config using `batch_size=2` and `grad_acc_steps=2`.
- `configs/teacher_student_recovery_v100_smoketest.yaml`: same V100-oriented setup, but wired to the toy dataset generated under `postprocessing/data/dataset`.

## Dataset Preparation

Place input videos under `postprocessing/data/` and run:

```bash
/mnt/models/orbis/.venv/bin/python /mnt/models/orbis/postprocessing/prepare_video_dataset.py --overwrite
```

By default this writes outputs to `postprocessing/data/dataset/`:

- `train.h5`
- `val.h5`
- `train_hdf5_paths.txt`
- `val_hdf5_paths.txt`
- `val_samples.json`
- `dataset_metadata.json`

The generated HDF5 layout matches the existing datasets in `data/custom_multiframe.py`.

## Training Entry Point

Training still runs through the repo's standard launcher:

```bash
/mnt/models/orbis/main.py
```

That is intentional. `main.py` only instantiates the configured model and datamodule, then hands them to the Lightning trainer. The teacher-student logic itself lives inside `TeacherStudentRecoveryModelIF` in `student_teacher_recovery.py`.

In other words:

- `main.py` is the generic trainer entrypoint
- `TeacherStudentRecoveryModelIF` is the actual teacher-student recovery model
- the teacher is frozen
- the student is the pruned model
- the loss combines the normal flow-matching objective with teacher distillation losses

## Smoke Test On V100

The smoke-test config is already wired to the generated dataset manifests:

- `postprocessing/data/dataset/train_hdf5_paths.txt`
- `postprocessing/data/dataset/val_samples.json`

Run a one-epoch smoke test with:

```bash
WM_WORK_DIR=/mnt/models/orbis/logs_wm \
/mnt/models/orbis/.venv/bin/python /mnt/models/orbis/main.py \
  -b /mnt/models/orbis/postprocessing/configs/teacher_student_recovery_v100_smoketest.yaml \
  --name teacher_student_smoketest_v100 \
  --num_epochs 1
```

This setup has already been smoke-tested on a `Tesla V100-SXM2-32GB` with:

- `batch_size=2`
- `grad_acc_steps=2`
- peak reserved memory around `23.4 GiB` during the synthetic/real smoke checks

`batch_size=4` did not fit on the V100 for the current teacher-student setup.

## Full Recovery Training

For a real V100 run, start from:

```bash
WM_WORK_DIR=/mnt/models/orbis/logs_wm \
/mnt/models/orbis/.venv/bin/python /mnt/models/orbis/main.py \
  -b /mnt/models/orbis/postprocessing/configs/teacher_student_recovery_v100.yaml \
  --name teacher_student_recovery_v100 \
  --num_epochs 3
```

Before that, replace the placeholder dataset paths in `configs/teacher_student_recovery_v100.yaml`.

For the toy dataset generated by `prepare_video_dataset.py`, use the smoke-test config instead.

## Distillation Setup

The current recovery model supports:

- output distillation via `output_distill_weight`
- feature distillation via `feature_distill_weight`

By default feature distillation is applied every fourth block plus the last block.

The current stage-1 trainable subset is:

- `space_mlp`
- `time_mlp`
- `adaLN_modulation`
- `adaLN_time_attn_modulation`
- `final_layer`
- `t_embedder`
- `frame_emb`

The tokenizer remains frozen.

## Notes

- The student and teacher use deterministic context handling during distillation to avoid stochastic context-drop mismatches.
- The smoke-test config is intended for wiring validation, not model quality evaluation.
- If you move the teacher or student runs away from the default directories, override the run or checkpoint paths in the model config.
